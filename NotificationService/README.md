The Notification Service is responsible for handling notifications in the e-commerce system. It listens for events generated by other services, such as user registration or order creation, and processes them to generate notifications.

This service is built using Spring Boot and integrates with Apache Kafka to consume event messages published by other services. Instead of exposing many REST APIs, the Notification Service primarily works in an event-driven manner, listening to Kafka topics and reacting to events such as user creation or order completion.

To run the Notification Service, ensure that Eureka Server and Kafka are running. The service can then be built and started using Maven.

mvn clean install
mvn spring-boot:run

Once started, the service registers with Eureka and begins consuming events from the configured Kafka topics.
